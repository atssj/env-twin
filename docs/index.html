How It Works

env-twin follows a simple yet robust process to keep your environment files safe and synchronized.

1

Discovery & Analysis

Scans your project for .env* files and reads your source of truth (defaults to .env) to identify all variable keys, values, and comments. It understands complex file structures and preserves your formatting.

2

Sanitization

For .env.example files, it automatically replaces sensitive values with safe placeholders (e.g., input_API_KEY). This ensures you never accidentally commit secrets to your repository while keeping your templates up to date.

3

Safe Synchronization

Creates a timestamped backup in .env-twin/ before updating target files. It only adds missing keys, never overwriting your existing local values. If anything goes wrong, you can easily restore from a backup.

Quick Start

Run env-twin to sync your environment files using the default configuration:

npx env-twin

Or customize source and destination paths:

npx env-twin --source .env.development --dest .env.dev.example

Available Commands

sync

Synchronize environment variable keys across all .env* files with automatic backup.

npx env-twin sync

restore

Restore .env* files from backups with rollback support and interactive selection.

npx env-twin restore

clean-backups

Remove old backups to save disk space while keeping the most recent ones.

npx env-twin clean-backups
